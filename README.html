Scaffold

Agent-driven executive function system. Signal conversation, desktop UI, background cortex, and cross-agent session bus — all sharing one SQLite brain.

Signal Agent Desktop UI Cortex Runtime Session Bus SQLite Brain

Architecture

Signal ──→ Agent (tool-use loop) ──→ Brain (SQLite) ↑ Web UI ──→ API server ──────────────────┘ ↑ Cortex ──→ Background scheduler ────────┘ ↑ Session Bus ──→ Cross-agent messaging ──┘ ↑ Coder ──→ Chain runner (claude subprocs) ──┘

Current Status

2A Foundation — config loader, conversation log, suppression model, mutation APIs
2B Agent rewrite — tool-use conversational loop + config-driven prompts/tools
2C Cortex runtime — singleton scheduler + bulletin cache + boot wiring
2D Inbox/desktop wiring — confirm/override/archive endpoints + frontend actions
2E Maintenance tasks — prioritize, session cleanup, consolidation, decay, prune, reindex
CAL Google Calendar integration — OAuth2 flow, event fetching, agent tool
LIFE LifeOS — Goals (binary/measurable/habit), tasks (recurring), notes, domain-scoped
DASH Dashboard — calendar panel, domain health, completions, goals overview, task list
NOTE Notebooks — domain-scoped views with goals/tasks/notes tabs
SRCH Search — global search across memories, goals, tasks, notes
BUS Session Bus — cross-agent messaging with TTL, queues, long-poll
CODE scaffold-coder — chain runner (scout→planner→worker→reviewer), SSE live stream, #/coder UI, dispatch_code_task agent tool
3A Hybrid memory search — FTS5 + vector retrieval with score fusion

LLM Routing

Provider/model selection is controlled by config/llm.yaml using providers, profiles, and routes.

RoutePurpose
brain.respondSignal conversation reply loop (tool-use)
brain.triageCapture triage JSON classification
brain.prioritizeDesk prioritization JSON generation
cortex.bulletinMemory bulletin synthesis
cortex.semanticConsolidation decisions
cortex.observationsObservation extraction
Example: Mixed providers (Anthropic + Ollama) 
profiles:
  respond_default:
    provider: anthropic_main
    model: claude-haiku-4-5
  ollama_semantic:
    provider: ollama_specialist
    model: qwen2.5:14b-instruct

routes:
  brain.respond:
    profile: respond_default
  cortex.semantic:
    profile: ollama_semantic
    lock_provider: true

scaffold-coder

Goroutine inside the daemon that picks up code_task messages from the session bus and runs chains of claude subprocesses. One chain at a time (semaphore=1); bus queues extras naturally.

ChainStepsUse when
implementscout → planner → worker → reviewer"implement issue #N", "build feature X"
fixworker → verifySmall bug fix, obvious change
specscout → planner"spec out X" — plan only, no code
singleworkerRaw one-shot escape hatch

Run Directory

/tmp/scaffold-coder/{uuid}/
  task.md          ← original task
  status.json      ← {chain, status, started_at, ended_at, …}
  steps/
    1-scout/
      prompt.md    ← rendered prompt (skill + template)
      events.jsonl ← append-only claude stream events
      output.md    ← step output → fed to next step
    2-planner/ …
    3-worker/  …
    4-reviewer/ …

Session Bus Protocol

# Signal agent → coder
{ "type": "code_task", "task": "implement goal recurrence",
  "chain": "implement", "cwd": "/home/mikekey/Builds/scaffold",
  "reply_to": "scaffold-agent" }

# Coder → Signal agent on completion
{ "type": "coder_result", "task_id": "uuid", "chain": "implement",
  "status": "done", "summary": "PR #48 opened", "elapsed_s": 422 }

SSE Events (GET /api/coder/stream)

event: chain_started  data: { task_id, chain, task, steps }
event: step_started   data: { task_id, step, step_num, step_total }
event: step_event     data: { task_id, step, type, tool?, input? }
event: step_done      data: { task_id, step, elapsed_s }
event: chain_done     data: { task_id, status, summary, elapsed_s }
event: chain_failed   data: { task_id, failed_step, error, elapsed_s }
